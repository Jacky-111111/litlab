Main Cursor Prompt
You are my senior full-stack engineering partner. Help me build a portfolio-ready MVP for my capstone project.

Project name: LitLab

Goal:
Build a beginner-friendly AI-powered research assistant web app. The app should help students start and manage research projects, search and save papers, understand papers with AI, and get structured guidance for different research writing approaches.

Tech stack requirements:
- Frontend: HTML, CSS, vanilla JavaScript
- Backend: FastAPI (Python)
- Database/Auth: Supabase
- Deployment target: Vercel
- AI: OpenAI API
- Style: modern, clean, beginner-friendly, portfolio-ready
- Color palette: blue + black/white, minimal and elegant

Core product concept:
LitLab is an AI research assistant for beginners. It should help users:
1. Analyze existing papers with AI and generate comprehension questions
2. Manage research projects in a structured way with accounts and a database
3. Search and save papers from academic APIs, then recommend related reading
4. Guide users through research frameworks such as:
   - IMRAD
   - Review / Survey
   - Theoretical Paper
   - Case Study

MVP scope:
Do NOT try to build everything. Focus on a clean, working MVP with these features:

1. Authentication
- Sign up / log in / log out with Supabase Auth
- Protect user-specific data

2. Project management
- Users can create, edit, delete, and view research projects
- Each project has:
  - title
  - description
  - research type / framework
  - created_at
  - updated_at

3. Research framework guidance
- When a user creates a project, they choose one of:
  - IMRAD
  - Review / Survey
  - Theoretical Paper
  - Case Study
- Show a structured checklist / guided sections for each type
- This guidance should be thoughtfully designed and easy for beginners to follow

4. Paper search
- Add a search bar that searches academic papers using a public academic API
- Prefer a stable API like Semantic Scholar, OpenAlex, or arXiv
- Display results with:
  - title
  - authors
  - year
  - abstract snippet
  - source
  - external link if available

5. Save papers to a project
- Users can save papers from search results into a selected project
- Each saved paper should store basic metadata in Supabase

6. AI paper understanding
- For a saved paper, users can click:
  - “Summarize”
  - “Explain for a beginner”
  - “Generate quiz questions”
- Use OpenAI API for these features
- Quiz questions should test understanding, not just restate the abstract

7. Related paper recommendations
- For MVP, simple recommendation logic is enough
- Example: use keywords from the saved paper title/abstract to fetch related papers
- Keep this simple and explainable

Non-goals for MVP:
- No mobile app yet
- No Chrome extension yet
- No real-time collaboration yet
- No advanced paper editing workflows yet
- No giant feature creep

Important engineering constraints:
- Keep architecture simple and explainable for an oral exam
- Write clean, readable code
- Avoid overengineering
- Include comments for important logic
- Use environment variables for all secrets
- Never hardcode API keys
- Handle errors gracefully
- Make the UI polished enough for a portfolio demo
- Make the app understandable for a beginner developer

Please do the following in order:

STEP 1: Plan
First, give me:
- a concise technical architecture
- recommended folder structure
- database schema for Supabase
- API endpoint list
- a clear MVP implementation plan in phases

STEP 2: Scaffold
Then generate the initial project structure for:
- frontend/
- backend/
- shared docs if needed

STEP 3: Backend
Implement FastAPI backend routes for:
- auth-related integration helpers if needed
- projects CRUD
- paper search proxy endpoint
- save paper to project
- AI summarize / explain / quiz endpoints

STEP 4: Frontend
Implement a clean vanilla HTML/CSS/JS frontend with:
- landing page
- auth pages or auth modal
- dashboard
- project detail page
- paper search UI
- saved papers list
- AI analysis panel

STEP 5: Styling
Use a modern blue + black/white design system:
- white or very light background
- deep blue accents
- black / dark gray text
- rounded cards
- subtle shadows
- clean buttons
- responsive layout

STEP 6: Docs
Generate:
- README.md
- .env.example
- setup instructions for local development
- deployment notes for Vercel
- Supabase setup instructions

Code quality rules:
- Keep functions small and readable
- Name variables clearly
- Avoid clever but confusing abstractions
- Explain any non-trivial logic
- Structure the code so I can explain it in an oral exam
- Prefer clarity over complexity

Product tone:
LitLab should feel:
- smart
- trustworthy
- academic but not intimidating
- friendly to beginners
- polished enough for a portfolio

Deliverables:
- full MVP code
- setup instructions
- clear explanation of where I should put my environment variables
- notes on what parts I should manually customize

When making implementation choices, prefer the simplest robust solution.
Start with STEP 1 only.
